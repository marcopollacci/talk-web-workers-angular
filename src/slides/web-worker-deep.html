🤝🏻 Shared Worker

Dedicated Workers are the most common: They are created from a single page with new Worker(). Each instance is independent (if you open two tabs of the same app, you will have two distinct Web Workers). Perfect for operations tied only to the current page (e.g., file parsing or heavy calculations).

🤝🏻 Shared Worker

  • 🌍 Can be shared across multiple tabs of the same origin
  • 😎 Keep alive until all tabs are closed (even if the page is reloaded)
    •
  • 💬 Communicate through the MessagePort API
    •

MessagePort API: https://developer.mozilla.org/en-US/docs/Web/API/MessagePort : similar to postMessage but more powerful and flexible

🤝🏻 Shared Worker - Practical Use Cases

  • 🗣️ Cross-tab communication through the MessagePort API
  • 🗃️ Shared Cache or Data Storage
    •
  • 👥 Session Management
    •

Let's Code 👨‍💻

  
    const clients = [];
    
    self.onconnect = (event) => {
      const [port] = event.ports;
      clients.push(port);
      port.postMessage("Worker connected");
      port.onmessage = (e) => {
        clients.forEach((client) =>
          client.postMessage(
            `New client connected: ${clients.length} with message: " ${e.data}"`
          )
        );
      };
    };

When the client disconnects (closing the tab or window), the onclose event is triggered.

The worker removes the client from the list and broadcasts a "Client disconnected" message to all remaining clients.

Some limitations ❌
  • ❌ No access to the DOM
  • ❌ No access to global variables like window, document, or parent
  • ❌ No synchronous APIs like alert(), confirm(), or prompt()
  • ❌ No access to storage APIs
    •
  • ❌ Some APIs like localStorage, sessionStorage, and cookies are not accessible. IndexedDB is only partially available.
  • ❌ No synchronous import of ES modules - Modules must be imported explicitly and asynchronously (importScripts() works only in classic workers, not module workers).
  • ❌ Restricted access to certain JavaScript APIs - Web Workers cannot use WebRTC, CanvasRenderingContext2D (except OffscreenCanvas), some security-related APIs, and other browser-specific features.
